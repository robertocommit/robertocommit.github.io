HandsON
This page (currently under development) has the goal to illustrate what Roberto is able to achieve practically speaking, giving for each activity a brief description. All these skills have been learnt during previous work experiences, but also building personal projects or experiments.
BI Audit
The way an organisation implement and structure a BI tool is one of the first signals about potential inefficiencies. At a certain level, from inside an organisation it is not possibile to understand opportunities and limitation, this is why having a professional looking at the tool from different angles it is extremely valuable.


BI Maintenance
BI tools require regular maintenance. From managing permissions to relocate queries, the number of daily hidden small tasks necessary to keep a system healty and up to date is often underestimate. Any organisation would require a specific role to take care of these aspects, but it is not always the case. Externalising this activity is a brave decision able soon pay to high dividends.


Query Optimisation
Nobody likes slow queries. Data needs to be deliver fast to enahnce decisions and move business forward. Sometimes queries can be written better, while in other cases tables' structure can be optimised allowing faster results. Moreover, queries' style must be consistent, allowing a smart reuse of the code already written.


Crawling
Any company faces the need to extract data from the web. For a multitude of reasons, having the possibility to acquire data previously not available can lead to competititive advantages. Data are out there but often not easy to access, because of time constrains or technical limitations. Rely on reilable and scalable web crawers to bring your data to a next level.


Features Engineering
The biggest part of a machine learning project is the iterative process of features defining and features improvement. This activity can be considered boring from a data scientist point of view, because not directly connected with algorithms behind the learning pipeline. However, this part is crucial and important as the algorithm itself if not even more.


Automation
This section does not need presentation. A digital business not able to automise processes will face sooner or later big troubles. Automate from the day one in order to keep focusing on what really matters: your business.


Marketing Automation
Marketing is probably the department where automation can be more significant and disruptive. Indeed, marketing is also one of the biggest cost cente. Specific skills and talent are required to drive leads to a company, that's why time cannot be wasted dealing with data in unproductive ways. Free your marketing department from non value added activities.

KPIs Audit
KPIs are at the core of a business. Theoretically any process would require to be monitored by a KPI. There is however a bad habit to spend more time defining and building KPIs then using it. A wise decision is often to make a step back.


Analysis
Diving into data with the goal to analyze them is a must have skill for any person working in the digital industry. However, things get easily more complex and articulated. Specific tools and techniques are required to provide puntual, correct and reprudicible analysis.


Reporting
In the last ten years the activity of reporting has known an immense switch due to the rise of web technologies. Nowdays is not accattable anymore to rely on a person to receive a report at the same hour on a daily basis. It is never to late to modernize your reporting structure.


Dashboard
A dashboard should not only be considered as data, queries and charts. A dashboard is a story, combining design, aesthetics, UX and UI. Details are essential, as colors, backgrounds, fonts, sizes, titles, positions, headers, alignments, approachability, efficiency and engagement. A dashboard is a real product with real use cases and real users.


API Development
API endpoints are critical for a business, because they allow external players to interact with internal data, but also to ensure a fluid flow of information within internal systems. Standardise the creation of API endpoints with modern and light technologies, while having always in control costs and performances.


API Reverse Engineering
Activity strictly connected with web crawling, but also with computer security. Without dealing with espionage or unethical things, API reverse engineering is necessary for example because documentation is not properly done, so there is the need to self understand how things work.


Scripting and tools
Digital business' Necessities are so variagate and colourful that are not possible to be categorised under a single umbrella. Any kind of small script and tool can save precious time, especially if an activity must be repeated multiple time. It is important to have a person in a team able to make colleagues' life easier and more pleasant.


Sheets Automation
For years many have tried to get rid of sheets, but the true is that they are here to stay. Moreover, sheets are getting better and better, making them easier to use, more intuitive and programmable. There are people with the talent to make sheets looking beauty and sexy, it is a luck to work closely with such a profile.


Notification
As anything else, also notifications can be pleasant to be received and to be consumed if properly designed. It is not time anymore to spam around unreadable bad formatted messages, considering the incredibile technologies available today for this purpose.


ETL
Extract, transform and load is the base of modern data engineering. Any company has the need to build reilable data pipelines, interating with API endpoints as well as databases. This activity if properly done allows a business to scale without perceving downtimes or inconsistencies.
