HandsON
This page (always under development) has the goal to illustrate what Roberto is able to achieve practically speaking, giving for each activity a brief description. All these skills have been learnt during work experiences, but also building personal projects or experiments.
Datalization
This word officially does not exists, and it is often used in conjunction with digitalization. It gives a powerful understanding of what companies, which are transitioning from old managament paradigms to new ones, are looking for to survive. Datalizations brings digitalisation. A company able to rely on a strong and flexible data structure is a company that can take data drive decisions.
BI Audit
The way an organisation implements and structures a BI tool is one of the first signals about potential inefficiencies. At a certain point within an organisation it is not possible anymore to understand opportunities and limitations. Having a professional looking at the tool from different angles can be extremely valuable.
BI Maintenance
BI tools require regular maintenance. From managing permissions to relocate queries, the number of daily hidden small tasks necessary to keep a system healthy and up to date is often underestimated. Any organisation would require a specific role to lead initiatives on these aspects, but is not always the case, producing during time outdated and fragile systems.
Query Debug
Queries are getting slower, or presenting wrong results, and nobody knows why. With patience and dedication, even the most complex query can be divided into small pieces of code, making it easier to manage, understanding what is not working, and most importantly, why is not working.
Query Optimisation
Nobody likes slow queries. Data must be fluid to enhance decisions and move business forward. Sometimes queries can be written better, while in other cases tables' structure can be optimised. Moreover, queries' writing style must be consistent, allowing a smart reuse of the code already written.
Crawling
Any company faces the need to extract data from the web. For a multitude of reasons, having the possibility to acquire data previously not available can lead to competitive advantages. Data are out there, but frequently not easy to access, because of time constraints or technical limitations. Reliable and scalable web crawlers are essential parts of any technology stack.
Features Engineering
The biggest part of a machine learning project is the iterative process of features defining and features improvement. This activity can be considered boring from a data scientist point of view, because not directly connected with algorithms behind the learning pipeline. However, it is crucial and important as the algorithm itself if not even more.
Automation
This section does not need presentation. A digital business not able to automise processes will face sooner or later big troubles. Automate from day one in order to keep focus on what really matters: your business.
Marketing Automation
Marketing is probably the department where automation can be more significant and disruptive. Indeed, it is also one of the biggest cost centre. Specific skills and talent are required to drive leads to a company, that's why time cannot be wasted dealing with data in unproductive ways.
KPIs Audit
KPIs are at the core of a business. Theoretically, any process would require monitoring by a KPI. There is however a bad habit to spend more time defining and building KPIs than using it. A wise decision is often to make a step back, simplifying the structure to take control.
Analysis
Diving into data with the goal to get insights out of them is an essential skill for anyone working in the digital era. However, things easily get too complex and articulated. Specific tools and techniques are required to provide punctual, correct and reproducible analysis.
Reporting
In the last ten years the activity of reporting has known an immense switch due to the rise of web technologies. It is never too late to start modernizing your reporting structure.
Dashboard
A dashboard should not only be considered as data, queries and charts. A dashboard is a story, combining design, aesthetics, UX and UI. Details are essential, as colors, backgrounds, fonts, sizes, titles, positions, headers, alignments, approachability, efficiency and engagement. A dashboard is a real product with real use cases and real users.
API Development
API endpoints are critical for a business, because they allow interaction with external players, as well as ensuring a fluid flow of information within internal systems. Here the necessity to standardise the creation of API endpoints with modern and light technologies, while having always in control costs and performances.
API Reverse Engineering
Activity strictly connected with web crawling, but also with computer security. Without dealing with espionage or unethical things, API reverse engineering can be necessary for example when documentation is not properly done, or to QA software. There is the often the need to self understand how things work behind the scenes.
Scripting and tools
Digital businesses' necessities are so variegated and colourful that is not possible to be categorised them under a single umbrella. Any kind of small script or tool can save precious energies, especially if an activity must be repeated multiple times.
Sheets Automation
For years many have tried to get rid of sheets, but the truth is that they are here to stay. Moreover, sheets are getting better, making them easier to use, more intuitive and programmable. There are people with the rare talent to make sheets a pleasant experience. It is a luck to work closely with such a profile.
ETL
Extract, transform and load is the core of modern data engineering. Any company has the need to build reliable data pipelines, interacting with API as well as databases. This activity must grant flexibility to scale without perceiving downtimes or inconsistencies, facilitating the growth of a business.
ETL QA
Once an ETL pipeline is ready, it is time to test if outputs are correct. This activity is extremely important and can be achieved using proper techniques and technologies. Those tests often not only prove correctness, but are also a vital source of improvements and new ideas.
Notification
As anything else, notifications if properly designed can be pleasant experience if properly managed. It is not time anymore to spam around unreadable bad formatted messages, considering the incredibles technologies available today for this exact purpose.
