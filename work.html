HandsON
This page (steadily under development) has the goal to illustrate what Roberto is able to achieve practically speaking, giving for each activity a brief description. All these skills have been learnt during previous work experiences, but also building personal projects or experiments.
BI Audit
The way an organisation implement and structure a BI tool is one of the first signals about potential inefficiencies. At a certain level, from inside an organisation it is not possible anymore to understand opportunities and limitations. This is why having a professional looking at the tool from different angles cane be extremely valuable.
BI Maintenance
BI tools require regular maintenance. From managing permissions to relocate queries, the number of daily hidden small tasks necessary to keep a system healthy and up to date is often underestimated. Any organisation would require a specific role to take care of these aspects, but it is not always the case, leading during time to outdated systems.
Query Debug
Queries are getting slower, or they are presenting wrong results, and you do not know why. With patience and dedication, even the most complex query can be divided into small pieces of code, making easier to debug it understanding what is not working, and most importantly, why.
Query Optimisation
Nobody likes slow queries. Data needs to be delivered fast to enhance decisions and move business forward. Sometimes queries can be written better, while in other cases tables' structure can be optimised allowing faster results. Moreover, queries' style must be consistent, allowing a smart reuse of the code already written.
Crawling
Any company faces the need to extract data from the web. For a multitude of reasons, having the possibility to acquire data previously not available can lead to competitive advantages. Data are out there, but often not easy to access, because of time constraints or technical limitations. Rely on reliable and scalable web crawlers to bring your data to a next level.
Features Engineering
The biggest part of a machine learning project is the iterative process of features defining and features improvement. This activity can be considered boring from a data scientist point of view, because not directly connected with algorithms behind the learning pipeline. However, this part is crucial and important as the algorithm itself if not even more.
Automation
This section does not need presentation. A digital business not able to automise processes will face sooner or later big troubles. Automate from the day one in order to keep focus on what really matters: your business.
Marketing Automation
Marketing is probably the department where automation can be more significant and disruptive. Indeed, marketing is also one of the biggest cost centre. Specific skills and talent are required to drive leads to a company, that's why time cannot be wasted dealing with data in unproductive ways.
KPIs Audit
KPIs are at the core of a business. Theoretically, any process would require monitoring by a KPI. There is however a bad habit to spend more time defining and building KPIs than using it. A wise decision is often to make a step back.
Analysis
Diving into data with the goal to analyze them is a must have skill for any person working in the digital industry. However, things easily get too complex and articulated. Specific tools and techniques are required to provide punctual, correct and reproducible analysis.
Reporting
In the last ten years the activity of reporting has known an immense switch due to the rise of web technologies. Nowadays is not acceptable anymore to rely on a person to receive a report at the same hour on a daily basis. It is never too late to modernize your reporting structure.
Dashboard
A dashboard should not only be considered as data, queries and charts. A dashboard is a story, combining design, aesthetics, UX and UI. Details are essential, as colors, backgrounds, fonts, sizes, titles, positions, headers, alignments, approachability, efficiency and engagement. A dashboard is a real product with real use cases and real users.
API Development
API endpoints are critical for a business, because they allow interaction with external players, but also to ensure a fluid flow of information within internal systems. Standardise the creation of API endpoints with modern and light technologies, while having always in control costs and performances.
API Reverse Engineering
Activity strictly connected with web crawling, but also with computer security. Without dealing with espionage or unethical things, API reverse engineering can be necessary because documentation is not properly done, so there is the need to self understand how things work.
Scripting and tools
Digital business' necessities are so variegated and colourful that is not possible to be categorised under a single umbrella. Any kind of small script and tool can save precious time, especially if an activity must be repeated multiple times.
Sheets Automation
For years many have tried to get rid of sheets, but the truth is that they are here to stay. Moreover, sheets are getting better, making them easier to use, more intuitive and programmable. There are people with the talent to make working with sheets a pleasant experience, it is a luck to work closely with such a profile.
Notification
As anything else, notifications if properly designed can be pleasant to be received and to be consumed. It is not time anymore to spam around unreadable bad formatted messages, taking into account the incredibles technologies available today for this exact purpose.
ETL
Extract, transform and load is the core of modern data engineering. Any company has the need to build reliable data pipelines, interacting with API endpoints as well as databases. This activity must grant to a business the flexibility to scale without perceiving downtimes or inconsistencies.
ETL QA
Once an ETL pipeline is done, it is time to test if outputs are correct. This activity is extremely important and can be achieved using proper techniques and technologies. A good practice is to let evaluate the correctness of an ETL not to the person who has written it.
